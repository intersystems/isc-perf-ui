Parameter Explanations

  • UnitTestRoot: The root directory for unit tests.
  • Coverage Level: The level of coverage detail required (0-3).
    • 0 to track code coverage overall
    • 1 to track it per test suite (the default)
    • 2 to track it per test class;
    • 3 to track it per test method
  • Coverage Classes: The classes to include in the coverage, comma separated list.
  • Coverage Routines: The routines to include in the coverage, comma separated list.
  • PidList: List of process IDs to track execution from.
  • Timing: Whether to include timing data (0 or 1, 1 is to include).
